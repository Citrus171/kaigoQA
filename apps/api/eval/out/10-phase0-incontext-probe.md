# 10 Phase 0: in-context 参照実験（reasoning 失敗の真因判定）

決定日: 2026-06-17
目的: reasoning 5.9%（1/17 good）の失敗が「知識アクセス不足(a)」か「4B推論能力限界(b)」かを切り分ける。

## 実験手法

1. 対象: `rejudge-judge-openai-gpt-4o-e2e-ollama-gemma3-4b.jsonl` の answerSource=reasoning かつ edge good=false の 16件から、referencePoints（承認済み参照要点）を持つ 11件
2. 各 query に referencePoints を system prompt へ in-context 注入し、gemma3:4b で回答を再生成
3. GPT-4o（OpenRouter）で再採点。good = factual && sufficient && !overreach
4. baseline（参照なし 0%）と比較

## 結果

| 条件 | good率 | n |
|---|---|---|
| baseline（参照なし） | 0/11 = **0.0%** | 11 |
| in-context 参照あり | 3/11 = **27.3%** | 11 |
| 改善 | +3件（+27.3pt） | |

## 改善/非改善の内訳

### good になった（知識アクセスで解決、3件）

| id | query | judge所見 |
|---|---|---|
| gold-18 | 要介護2でデイサービスは何回利用できますか | ケアプランに基づく説明が適切 |
| gold-32 | 医療費控除の対象になる介護サービスを教えてください | 医療費控除対象を正確に説明 |
| gold-33 | 限度額管理期間をまたぐ場合の利用回数の考え方は？ | 限度額の説明が正確 |

→ 事実列挙型の質問（知識を箇条書きで答えれば十分なもの）は参照注入で改善。

### good にならなかった（推論能力限界、8件）

| id | query | judge所見 |
|---|---|---|
| gold-19 | 負担割合証が2割になる条件 | 制度条件の誤りと不十分な回答 |
| gold-20 | 特定入所者介護サービス費の対象 | 具体的な条件説明が不足 |
| gold-24 | 入浴介助加算IとIIの違いと算定要件 | 具体的要件の説明が不足 |
| gold-25 | 処遇改善加算の算定区分 | 具体的な計算方法が不足 |
| gold-26 | 支給限度額を超えた分の自己負担 | 具体的な計算方法が不足 |
| gold-27 | 介護保険法第8条の解釈 | 第8条の内容誤り |
| gold-28 | 高額介護サービス費の上限の判定 | 詳細な説明が不足している |
| gold-30 | 看取り介護加算の算定要件 | 具体的な要件に答えていない |

→ 算定・適用判断・条文解釈を要する質問（知識の組み合わせ・推論が必要）は参照注入しても不十分。

## 判定

**mixed: 知識アクセスも推論能力もボトルネックだが、推論能力が支配的（8/11=73%は依然失敗）。**

| 要因 | 該当件数 | 割合 |
|---|---|---|
| 知識アクセス不足（参照注入で解決） | 3件 | 27% |
| 推論能力限界（参照注入でも解決せず） | 8件 | 73% |

単純な事実列挙型（gold-18/32/33）は参照注入で gemma3:4b でも回答可能。しかし算定・適用判断・条文解釈を要する質問では、参照知識を与えても「具体的な説明が不足」「計算方法が不足」と判定される。これは 4B モデルの表現力・多段推論の限界。

### LoRA の位置づけ変更

Phase 0 の結果から、LoRA の価値は限定的:
- 3件（27%）の事実列挙型は RAG で同等以上に解決可能（訓練不要・更新容易）
- 8件（73%）の推論型は 4B の表現力限界であり、LoRA（小データ）では構造的に解決困難
- LoRA に価値が出るのは「granite3.2:8b が実質的に答えられるが表現が不適切」な場合のみ
