# 04 Agent 割り当て方針（HANDOFF 運用）

決定日: 2026-06-17 / 改定: 2026-06-17（Qwenレビュー誤報を受け方針変更）

## 割り当て（改定後）

| 工程 | 担当 | 役割 |
|---|---|---|
| 生成 | **DeepSeek V4 Pro**（単独） | HANDOFF を受け取り、成果物を一括生成 |
| 検算 | **スクリプト（Python等）** | 成果物の数値を生データと機械的に照合（`off.flipped` カウント等） |
| 最終承認 | 人間 | 検算結果を確認し、結論を確定 |

## 検算スクリプト（必須）

毎回の成果物確定前に以下を機械的に照合する:
- flip率 → `python3 -c "import json; ..."` で flip JSONL の `off.flipped` / `on.flipped` を直接カウント
- offload率 → E2E JSONL の `routing.predictedTier` を直接カウント
- good率 → rejudge JSONL の `isGoodAnswer` を直接カウント
- gold承認数 → routing-gold.jsonl の `answerReview=="approved"` を直接カウント

## 経緯

| 日時 | 判断 | 理由 |

## Qwen 非採用の根拠

2026-06-17 のレビュー実績:
- 指摘3件（flip率逆転 / gold-20 on値 / gold-26 on値）
- 実ファイル・生データとの突合で**3件すべて正しい数値だった** → Qwen が表の列を読み間違えたと推定
- 検算には人間またはスクリプトの方が正確

## 今後の評価方針

- DeepSeek V4 Pro が安定して正確な数値を出す限り、単独運用を継続
- 精度に疑義が出れば別モデルでの AB テストを実施
- 検算スクリプトは成果物ごとに使い捨てでよい（3〜5行の Python ワンライナーで十分）
